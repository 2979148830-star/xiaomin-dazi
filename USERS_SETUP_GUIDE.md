# 用户数据库设置指南

## 📋 概述
本文档指导您如何配置"校园灵魂搭子"小程序的用户数据系统。

## 🔧 步骤 1: 创建 users 集合

### 方法 A: 通过云开发控制台创建

1. 登录 [腾讯云开发控制台](https://console.cloud.tencent.com/tcb)
2. 选择你的云开发环境（环境 ID 以自己的控制台为准）
3. 点击左侧菜单 "数据库"
4. 点击 "新建集合"
5. 集合名称输入: `users`
6. 点击确认

### 方法 B: 通过云函数创建（推荐）

如果你有云函数开发权限，可以创建一个初始化脚本来创建集合：

```javascript
// cloudfunctions/createUsersCollection/index.js
const cloud = require('wx-server-sdk');

cloud.init({
  env: 'your-cloud-env-id'
});

exports.main = async (event, context) => {
  const db = cloud.database();
  const _ = db.command;
  
  try {
    // 创建 users 集合
    // 注意: 云开发通常自动创建集合，无需手动创建
    
    return {
      success: true,
      message: 'users 集合已就绪'
    };
  } catch (err) {
    console.error(err);
    return {
      success: false,
      error: err
    };
  }
};
```

## 📊 users 集合数据结构

每个用户文档将包含以下字段：

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| _openid | String | ✅ | 用户唯一标识（系统自动生成） | "oABC123..." |
| user_name | String | ✅ | 用户昵称 | "学习搭子小王" |
| user_avatar | String | ❌ | 用户头像（云存储文件ID） | "cloud://xxx/avatars/xxx.jpg" |
| gender | String | ✅ | 性别 | "男生" / "女生" / "男女都行" |
| age | Number | ✅ | 年龄 | 20 |
| hobbies | String | ❌ | 爱好（逗号分隔） | "篮球,绘画,音乐" |
| my_tag | String | ❌ | MBTI人设标签 | "雷厉风行的卷王" |
| my_mbti | String | ❌ | MBTI类型 | "INTJ" |
| create_time | ServerDate | ✅ | 创建时间 | db.serverDate() |
| update_time | ServerDate | ❌ | 更新时间 | db.serverDate() |

## 🧪 步骤 2: 测试流程

### 测试前准备

1. **确保已登录**
   - 打开微信开发者工具
   - 确认已通过 "获取 OpenID" 云函数
   - 查看控制台是否有 "成功获取 OpenID" 日志

2. **检查云开发初始化**
   - 打开 `app.js`
   - 确认环境 ID 已替换为自己的云开发环境 ID
   - 检查 `traceUser: true` 是否已设置

### 测试步骤

#### 1. 进入"我的"页面
```
路径: /pages/mine/mine
```

#### 2. 查看初始状态
- 如果是新用户，应该看到:
  - 头像: 使用默认头像
  - 昵称: 空白
  - 性别: 请选择性别
  - 年龄: 空白
  - 爱好: 空白

#### 3. 更新个人资料
- 点击头像 → 选择一个头像
- 点击昵称输入框 → 输入昵称（如：测试用户）
- 点击性别 → 选择 "男生"
- 输入年龄 → 如：20
- 输入爱好 → 如：篮球、编程

#### 4. 保存资料
- 点击 "💾 保存个人信息" 按钮
- 应该看到提示: "保存成功"
- 查看云开发数据库中 `users` 集合，应该能看到新创建的文档

#### 5. 验证数据保存
在云开发控制台 → 数据库 → users 集合:
- 查看是否有新创建的文档
- 确认字段值正确:
  - `_openid`: 匹配当前用户
  - `user_name`: "测试用户"
  - `gender`: "男生"
  - `age`: 20
  - `hobbies`: "篮球、编程"
  - `my_tag`: "神秘校园散人"（如果之前已测过MBTI）
  - `my_mbti`: 如果之前已测过MBTI

### 测试发布页面

#### 1. 创建新帖子
```
路径: /pages/publish/publish
```

#### 2. 完成性格测试
- 完成3道题目
- 查看结果和标签

#### 3. 填写表单
- 选择活动类型
- 选择时间、日期、地点

- 填写电话号码

## 🔍 故障排查

### 问题 1: 无法保存用户信息

**症状**: 点击保存按钮后提示"保存失败"

**可能原因**:
- 云数据库权限未设置
- 用户记录已存在但更新失败

**解决方案**:
1. 检查云开发控制台 → 权限设置
2. 确保 `users` 集合的读写权限为"仅创建者可读写"或"所有用户可读写"
3. 查看控制台错误日志，确认具体错误信息

### 问题 2: 照片上传失败

**症状**: 选择头像后提示"头像上传失败"

**可能原因**:
- 云存储空间不足
- 云存储路径权限问题

**解决方案**:
1. 检查云开发控制台 → 云存储
2. 确认 `avatars/` 目录已创建且权限正确
3. 检查是否有足够的存储空间

### 问题 3: 发布后看不到性别和爱好标签

**症状**: 发布的帖子在广场和详情页都看不到性别/爱好标签

**可能原因**:
- `creator_gender` 或 `creator_hobbies` 字段未正确保存
- WXML 模板条件判断有误

**解决方案**:
1. 检查云开发数据库 → posts 集合
2. 确认帖子文档中包含这两个字段
3. 检查微信开发者工具的控制台是否有错误日志
4. 重新发布一个测试帖子

### 问题 4: "我的"页面显示旧数据

**症状**: 更新资料后重新进入"我的"页面，数据显示没有变化

**可能原因**:
- `onShow()` 未正确触发
- 全局数据缓存未更新

**解决方案**:
1. 退出小程序，重新进入
2. 检查 `app.js` 中的 `queryUserInfo()` 是否在 `getOpenId()` 成功后调用
3. 查看控制台是否有 "用户信息已获取" 日志
