# 发布功能故障排查指南

## 🐛 常见问题及解决方案

### 问题 1: OpenID 未获取成功

**症状**: 提示"请稍候，正在获取用户信息..."

**可能原因**:
1. `quickstartFunctions` 云函数未部署
2. 云开发环境未正确配置
3. 用户未授权获取 OpenID

**解决步骤**:

1. **检查云函数**
   ```
   cloudfunctions/quickstartFunctions/
   ├── index.js
   └── package.json
   ```

2. **部署云函数**
   - 在微信开发者工具中右键 `quickstartFunctions` 文件夹
   - 选择 "上传并部署：云端安装依赖"

3. **检查环境 ID**
   - 打开 `miniprogram/app.js`
   - 确认环境 ID 已替换为自己的云开发环境 ID
   - 如果环境 ID 不同，请在云开发控制台查看

---

### 问题 2: 数据库权限设置错误

**症状**: 提示"权限不足，请检查数据库权限设置"

**可能原因**:
- `posts` 集合的写权限未设置
- 集合不存在

**解决步骤**:

1. **进入云开发控制台**
   - 微信开发者工具 → 云开发
   - 点击"数据库"

2. **创建 posts 集合**
   - 点击"添加集合"
   - 集合名称: `posts`
   - 点击"确定"

3. **设置集合权限**
   - 点击 `posts` 集合
   - 点击权限设置
   - 写权限: 选择"仅创建者可写" 或 "所有用户可写"
   - 读权限: 选择"所有用户可读" 或 "仅创建者可读"
   - **推荐配置**:
     - 读: 所有用户可读
     - 写: 仅创建者可写

4. **验证集合存在**
   - 检查是否能在数据库列表中看到 `posts` 集合
   - 点击集合查看是否有数据

---

### 问题 3: 云数据库未初始化

**症状**: 提示"云数据库未初始化"

**可能原因**:
- 云开发未开通
- 数据库服务未启用

**解决步骤**:

1. **开通云开发**
   - 微信开发者工具 → 云开发
   - 如果未开通，点击"开通"

2. **启用数据库服务**
   - 进入云开发控制台
   - 点击"数据库"
   - 确认数据库服务已启用

3. **检查数据库初始化**
   - 在开发者工具控制台运行:
     ```javascript
     const db = wx.cloud.database();
     console.log('数据库初始化:', db);
     ```

---

### 问题 4: 网络连接问题

**症状**: 提示"网络错误，请检查网络连接"

**可能原因**:
- 网络不稳定
- 微信开发者工具网络设置问题

**解决步骤**:

1. **检查网络连接**
   - 确保设备已连接网络
   - 尝试切换 WiFi 或使用 4G/5G

2. **检查开发者工具网络设置**
   - 微信开发者工具 → 详情 → 本地设置
   - 勾选"不校验合法域名、web-view(业务域名)、TLS 版本以及 HTTPS 证书"
   - 勾选"启用调试"

3. **重新编译小程序**
   - 点击"编译"按钮
   - 重新尝试发布

---

### 问题 5: 数据格式错误

**症状**: 提示数据库错误

**可能原因**:
- 字段类型不匹配
- 必填字段为空

**解决步骤**:

1. **检查必填字段**
   在发布页面确保填写了:
   - ✅ 搭子类型 (type)
   - ✅ 日期 (date)
   - ✅ 时间 (time)
   - ✅ 地点 (location)
   - ✅ 人数 (total_people)
   - ✅ 性别 (gender)
   - ✅ 年级 (grade)
   - ✅ 电话 (phone)

2. **查看控制台日志**
   - 在开发者工具控制台查看 `console.log('准备发布的数据:', postData);`
   - 确认所有字段都有值

---

### 问题 6: 环境变量不匹配

**症状**: 数据库操作失败，提示找不到集合

**可能原因**:
- 代码中的环境 ID 与实际云开发环境不一致

**解决步骤**:

1. **查看当前环境 ID**
   ```javascript
   // 在开发者工具控制台运行
   console.log('当前环境:', wx.cloud.getEnv());
   ```

2. **修改 app.js 中的环境 ID**
   ```javascript
   // miniprogram/app.js
   wx.cloud.init({
     env: '你的实际环境ID',  // 替换为实际的环境 ID
     traceUser: true,
   });
   ```

3. **获取环境 ID**
   - 进入云开发控制台
   - 查看环境详情
   - 复制环境 ID

## 🔧 调试技巧

### 1. 查看详细错误信息

每次发布失败都会弹出模态框，显示具体错误原因。请截图保存。

### 2. 检查控制台日志

在微信开发者工具中查看控制台输出:
- `准备发布的数据:` - 显示即将发送到数据库的数据
- `发布成功，文档ID:` - 显示成功发布的文档 ID
- `发布失败详情:` - 显示失败的具体原因

### 3. 手动测试数据库操作

在开发者工具控制台运行以下代码测试:

```javascript
// 测试数据库连接
const db = wx.cloud.database();
console.log('数据库实例:', db);

// 测试集合是否存在
db.collection('posts').limit(1).get({
  success: (res) => {
    console.log('集合存在，示例数据:', res.data);
  },
  fail: (err) => {
    console.error('集合不存在或访问失败:', err);
  }
});
```

### 4. 检查用户权限

```javascript
// 在开发者工具控制台运行
const app = getApp();
console.log('OpenID:', app.globalData.openid);
console.log('用户标签:', app.globalData.userTag);
console.log('用户MBTI:', app.globalData.userMbti);
```

## ✅ 验证发布功能正常

完成上述修复后，按照以下步骤验证:

1. **检查集合权限**
   - 数据库中是否有 `posts` 集合
   - 权限设置是否正确

2. **测试发布**
   - 填写完整的帖子信息
   - 点击发布按钮
   - 查看是否成功

3. **验证数据**
   - 进入数据库控制台
   - 查看 `posts` 集合
   - 确认帖子已成功保存
   - 检查字段是否完整

4. **检查首页**
   - 切换到"广场"页面
   - 确认帖子已显示在列表中
